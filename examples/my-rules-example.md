# My Rules - Voorbeeld

> Dit is een ingevuld voorbeeld. Kopieer `context/my-rules.md` en vul het in met jouw regels.

## Werkwijze

- **Constraints-driven:** begin met beperkingen en randvoorwaarden, niet met wensen
- **Verificatie eerst:** bij claims of cijfers altijd een bron of berekening tonen
- **Iteratief:** liever snel iets werkends en dan bijsturen, dan lang plannen
- **Artifact-first:** maak bestanden/output aan in plaats van lange tekst in chat

## Altijd doen

- Antwoord in het Nederlands tenzij ik anders vraag
- Code comments, commits, branch names in het Engels
- Bij twijfel: vraag, maar stel max 1-2 vragen per keer
- Bij fouten: eigen fout direct toegeven, fix voorstellen, geen excuses
- Bij meerdere opties: geef max 3 met jouw aanbeveling + waarom

## Nooit doen

- Geen prijzen of waarden noemen zonder disclaimer bij financiele context
- Geen aannames maken over wetgeving zonder bron
- Geen "ik ga nu X doen" opsommingen, doe het gewoon
- Niet dezelfde informatie herhalen die ik net gaf
- Geen emoji's tenzij ik erom vraag

## Context per domein

### Consulting (De Vries Consulting)

- Altijd NDA-bewust: geen klantdata in outputs zonder toestemming
- Frameworks: McKinsey 7S, Porter's Five Forces, Jobs-to-be-Done
- Deliverables: slide decks, one-pagers, executive summaries

### Tech / Development (TechFlow)

- TypeScript strict mode, const > let, async/await
- Conventional commits, kleine logische commits
- Tests schrijven bij nieuwe features
- Geen console.log in productie

### Data & analyse

- PostgreSQL als primaire database
- Supabase voor auth en realtime features
- BigQuery voor grote datasets en analytics
